------------------------------------------------------------------------------------------------------------------------
Install & Run:
  npm i -g privacy-pools-cli
  npm i -g github:0xmatthewb/privacy-pools-cli  (unreleased/source builds)
  privacy-pools status
  privacy-pools upgrade --check                (check npm for a newer installed release)
  npm run dev -- status                        (from source, no global install)
  privacy-pools completion --help                  (shell autocomplete setup)

------------------------------------------------------------------------------------------------------------------------
Quick Start:
  privacy-pools init
  privacy-pools init --recovery-phrase-file <downloaded-file> (load an existing account from a website export)
  cat <downloaded-file> | privacy-pools init --recovery-phrase-stdin (stdin alternative for loading an account)
  privacy-pools init --signer-only (finish setup or replace the signer key without changing the account)
  privacy-pools flow start 0.1 ETH --to 0xRecipient          (easy path: deposit now, withdraw later)
  privacy-pools flow start 100 USDC --to 0xRecipient --new-wallet  (easy path with a dedicated workflow wallet)
  privacy-pools pools                                          (browse available pools)
  privacy-pools deposit 0.1 ETH
  privacy-pools accounts --chain mainnet --pending-only        (poll ASP review; keep the same --chain until it disappears)
  privacy-pools accounts --chain mainnet                       (then confirm approved vs declined vs POA Needed)
  privacy-pools withdraw 0.05 ETH --to 0xRecipient --pool-account PA-1
  Transaction commands use your default chain (set during init).
  Public dashboards like pools/activity/protocol-stats default to CLI-supported mainnet chains.
  Use --include-testnets to include supported testnets, or --chain to scope one network.
  Accounts is wallet-dependent: use --chain to keep approval checks on the same network as the deposit.

  Deposits are reviewed by the 0xBow ASP before approval.
  Most deposits are approved within 1 hour, but some may take longer (up to 7 days).
  ASP approval is required for withdraw, including --direct. Ragequit is your
  self-custody guarantee, always available to publicly recover funds to the
  original deposit address. Declined saved easy-path workflows use
  'flow ragequit' as their canonical public recovery path, and operators can
  also choose it manually after the public deposit exists.

------------------------------------------------------------------------------------------------------------------------
Recovery Phrase and Signer Key:
  Recovery phrase  restores this Privacy Pools account and enables private withdrawals
  Signer key       pays gas and sends transactions (can be set later with init --signer-only)
  The signer key may come from the same wallet as your recovery phrase or from a separate key you control.
  Note: PRIVACY_POOLS_PRIVATE_KEY env var takes precedence over a saved key file.
  Exception: flow start --new-wallet creates and uses a dedicated per-workflow wallet instead of the configured signer.

------------------------------------------------------------------------------------------------------------------------
Workflow:
  1. init           Set up your Privacy Pools account and config (run once)
  2. flow start     Easy path: deposit now and save a later private withdrawal
  3. flow watch     Human path: stay attached through funding, approval, delay, and withdrawal
  4. flow status    Agent path: poll the saved workflow snapshot without mutating it
  5. flow step      Agent path: advance at most one saved-workflow step
  6. flow ragequit  Self-custody recovery for saved workflows (always available)
  7. pools          Manual path: browse available pools
  8. deposit        Manual path: deposit into a pool (ASP vetting fee shown before confirming)
  9. accounts       Manual path: poll pending review, then confirm approval status and balances
 10. migrate status Read-only legacy check on CLI-supported chains
 11. withdraw       Manual path: withdraw privately (once approved; fee shown before confirming)
 12. history        View transaction history
  *  status         Check setup and connection health (checks run by default)
  *  upgrade        Check npm for updates or upgrade this CLI
  *  activity       Public onchain feed (for your history, use 'history')
  *  ragequit       Self-custody guarantee. Publicly recovers funds to deposit address
  *  withdraw quote Check relayer fees before withdrawing
  'migrate status' is read-only. The CLI does not submit migration transactions; use the website for actual migration or website-based recovery.
  It only checks chains currently supported by the CLI; review beta or website-only legacy migration surfaces in the website.
  Human 'flow start --new-wallet' waits for funding automatically; '--agent' returns an awaiting_funding snapshot instead.
  In machine mode, this path requires '--export-new-wallet <path>' so the generated key is backed up first.
  Manual commands remain available for advanced control.

------------------------------------------------------------------------------------------------------------------------
Flow States:
  awaiting_funding                    Workflow wallet needs ETH/tokens to proceed.
  depositing_publicly                 Deposit transaction submitted, awaiting onchain confirmation.
  awaiting_asp                        Deposit confirmed, ASP is reviewing.
  approved_waiting_privacy_delay      Approved; privacy delay timer running.
  approved_ready_to_withdraw          Ready for private withdrawal.
  withdrawing                         Withdrawal transaction in flight.
  completed                           Private withdrawal succeeded.
  completed_public_recovery           Ragequit recovery completed.
  paused_poa_required                 Proof of Association needed before private withdrawal.
  paused_declined                     Deposit declined; use flow ragequit for public recovery.
  stopped_external                    Workflow stopped by external event.

------------------------------------------------------------------------------------------------------------------------
Ragequit / Public Recovery:
  privacy-pools ragequit ETH --pool-account PA-1
  privacy-pools flow ragequit latest

  Ragequit publicly recovers a Pool Account to the original deposit address.
  It is your self-custody fallback when ASP approval is unavailable, a saved
  flow is declined, the relayer minimum blocks a saved full-balance withdrawal,
  or you explicitly choose public recovery instead of waiting.

  This does not provide privacy for that Pool Account. Prefer withdraw or
  flow watch when the account can continue through the private relayed path.
  In the website this corresponds to the Exit path.

Global Options:
  -c, --chain <name>    Target chain (mainnet, arbitrum, optimism; testnets: sepolia, op-sepolia)
  -r, --rpc-url <url>   Override RPC URL
  -j, --json            Machine-readable JSON output
  --output <fmt>        Output format: table (default), csv, json, yaml, wide, name
  --no-color            Disable colored output (also respects NO_COLOR env var)
  -y, --yes             Skip confirmation prompts
  -q, --quiet           Suppress human-oriented stderr output
  -v, --verbose         Enable verbose/debug output (-v info, -vv debug, -vvv trace)
  --no-progress         Suppress spinners/progress indicators (useful in CI)
  --no-header          Suppress header rows in CSV and wide/tabular table output
  --agent               Alias for --json --yes --quiet (agent/automation mode)
  --timeout <seconds>  Network/transaction timeout (default: 30)
  --jmes <expr>        Filter JSON output with JMESPath syntax
  --jq <expr>          Compatibility alias for --jmes (not jq syntax)
  --template <template> Render structured output through a lightweight {{path.to.value}} template
  --web                 Open the primary explorer or portal link in your browser when available
  --no-banner           Disable ASCII banner
  --profile <name>     Use a named profile (separate wallet identity and config)

Environment Variables:
  PRIVACY_POOLS_HOME / PRIVACY_POOLS_CONFIG_DIR  Override the CLI config directory.
  XDG_CONFIG_HOME  Fallback config base. Used as $XDG_CONFIG_HOME/privacy-pools when no Privacy Pools override is set and no legacy ~/.privacy-pools directory exists.
  PRIVACY_POOLS_PRIVATE_KEY  Signer private key; takes precedence over the saved .signer file.
  PRIVACY_POOLS_CIRCUITS_DIR  Override the circuit artifact directory with a trusted pre-provisioned path.
  PRIVACY_POOLS_RPC_URL / PP_RPC_URL  Override the RPC endpoint for all chains.
  PRIVACY_POOLS_ASP_HOST / PP_ASP_HOST  Override the ASP endpoint for all chains.
  PRIVACY_POOLS_RELAYER_HOST / PP_RELAYER_HOST  Override the relayer endpoint for all chains.
  PRIVACY_POOLS_RPC_URL_<CHAIN> / PP_RPC_URL_<CHAIN>  Override the RPC endpoint for one chain, for example ARBITRUM or SEPOLIA.
  PRIVACY_POOLS_ASP_HOST_<CHAIN> / PP_ASP_HOST_<CHAIN>  Override the ASP endpoint for one chain.
  PRIVACY_POOLS_RELAYER_HOST_<CHAIN> / PP_RELAYER_HOST_<CHAIN>  Override the relayer endpoint for one chain.
  PRIVACY_POOLS_AGENT  Enable agent mode by default (equivalent to --agent).
  PRIVACY_POOLS_QUIET  Suppress human-oriented stderr output by default, matching --quiet.
  PRIVACY_POOLS_YES  Skip confirmation prompts by default, matching --yes.
  PRIVACY_POOLS_NO_PROGRESS  Suppress spinners/progress indicators by default, matching --no-progress.
  NO_COLOR  Disable colored output, matching --no-color.
  PP_NO_UPDATE_CHECK  Set to 1 to disable the update-available notification.

------------------------------------------------------------------------------------------------------------------------
Interaction Modes:
  Human mode (default): interactive prompts + readable output.
  Agent mode: --agent for structured JSON output, no prompts, no banners.
  Equivalent to --json --yes --quiet.

------------------------------------------------------------------------------------------------------------------------
Advanced Modes:
  Unsigned mode builds transaction payloads without signing or submitting.
  Requires init (recovery phrase) for deposit secret generation.
  Does NOT require a signer key. The signing party provides their own.
  Output includes from plus description. from is null when the signer is unconstrained, and set when the protocol requires a specific caller.
  --unsigned           (default) Wrapped in JSON envelope: { schemaVersion, success, ... }
  --unsigned tx        Raw transaction array: [{ from, to, data, value, valueHex, chainId, description }]
             Raw format skips the envelope. Intended for direct piping to signing tools.
  --dry-run    Validate and generate proofs without submitting.

------------------------------------------------------------------------------------------------------------------------
nextActions:
  Success JSON responses may include nextActions[] when the CLI has a low-ambiguity follow-up to recommend.
  Each entry includes command, reason, when, cliCommand, args?, options?, and runnable?.

------------------------------------------------------------------------------------------------------------------------
Runnable Semantics:
  runnable=true  The cliCommand is complete and can be executed as-is.
  runnable=false The action is a template; fill in missing user input first.

------------------------------------------------------------------------------------------------------------------------
Ordering:
  The first matching action is the highest-priority recommendation.
  Private/resume paths come first, required public recovery comes before optional public recovery, and template deposit/withdraw actions come last.

Common Uses:
  privacy-pools deposit --agent         Emits the canonical pending-review polling command.
  privacy-pools status --agent          Emits the canonical init/pools/accounts follow-up.
  privacy-pools tx-status <submissionId> --agent  Polls async write confirmation after --no-wait.
  privacy-pools flow status --agent     Emits saved-workflow poll, step, or ragequit guidance.

Profiles:
  Profiles let one machine keep separate Privacy Pools config, wallet state, and signer setup under different names.
  Use --profile <name> on any command to select one profile for that invocation.

------------------------------------------------------------------------------------------------------------------------
Commands:
  privacy-pools config profile list
  privacy-pools config profile create <name>
  privacy-pools config profile use <name>
  privacy-pools config profile active

------------------------------------------------------------------------------------------------------------------------
When To Use Them:
  Use profiles when you need separate operator identities, separate test/main workflows, or isolated machine-runner state on one host.

------------------------------------------------------------------------------------------------------------------------
Pool Accounts:
  A Pool Account (e.g. PA-1) is your onchain deposit.
  Approval, withdrawal, and ragequit happen per Pool Account, not per wallet.

  pending        Still under ASP review
  poa_required   Proof of Association is required before private withdrawal
  declined       Private withdrawal unavailable; ragequit remains available
  spent          Already withdrawn
  exited         Ragequit completed

------------------------------------------------------------------------------------------------------------------------
JSON Contract:
  Use --json for machine-readable output. Use --agent for --json --yes --quiet.
  Successful commands emit:
  { "schemaVersion": "2.0.0", "success": true, ...payload }
  Failed commands emit:
  { "schemaVersion": "2.0.0", "success": false, "errorCode": "...", "error": { ... } }

------------------------------------------------------------------------------------------------------------------------
Filtering:
  --json <fields>       Select top-level fields by comma-separated name.
  --json=<fields>       Inline form for compact scripts and automation.
  --jmes <expression>     Apply a JMESPath expression to the final envelope.
  --jq <expression>       Compatibility alias for --jmes (not jq syntax).
  --template <template>   Interpolate the final envelope with {{path.to.value}} placeholders and {{#items}}...{{/items}} list iteration.
  Unknown selected fields fail with INPUT_UNKNOWN_JSON_FIELD, availableFields[], and did-you-mean suggestions when available.
  Invalid JMESPath expressions fail before command output is emitted.
  --json <fields>, --jmes/--jq, and --template are mutually exclusive.

Modes:
  --yes       Skip confirmation prompts. Use only after the command inputs are fully reviewed.
  --dry-run   Preview validation and generated outputs without signing or submitting.
  --unsigned  Build transaction payloads without signing or submitting; implies --yes.
  --agent     Machine mode: --json --yes --quiet.

------------------------------------------------------------------------------------------------------------------------
Safety Notes:
  --dry-run and --unsigned may use approximate validation because no transaction is submitted.
  Fund-moving commands still surface warnings and privacy-cost manifests where relevant.
  Prefer relayed withdrawals. withdraw --direct is a stronger privacy hazard than ragequit.

------------------------------------------------------------------------------------------------------------------------
Quiet Mode:
  --quiet suppresses human-oriented stderr output while keeping machine-readable stdout intact.
  Use it in CI or scripts when you want less narration but do not want JSON mode.

------------------------------------------------------------------------------------------------------------------------
Status Special Case:
  privacy-pools status --quiet prints one concise stdout summary line instead of going fully silent.
  That keeps automation-friendly logs from looking like a hung process.

------------------------------------------------------------------------------------------------------------------------
Related Modes:
  --json keeps structured envelopes on stdout.
  --agent is shorthand for --json --yes --quiet.

------------------------------------------------------------------------------------------------------------------------
Troubleshooting:
  Stale data?      Commands auto-sync; force a full re-sync with 'privacy-pools sync'.
  ASP unreachable?  Check 'privacy-pools status' (health checks run by default).
  Long proof time?  Proofs use bundled circuit artifacts. The first proof may spend a moment verifying them.
  Native fallback?  Set PRIVACY_POOLS_CLI_DISABLE_NATIVE=1 to force the JS launcher,
                   or see docs/runtime-upgrades.md for runtime troubleshooting and overrides.
  Upgrade path?     Run privacy-pools upgrade to check npm for updates or upgrade this CLI.
  Not approved?     Deposits are reviewed by the ASP.
                   Most deposits are approved within 1 hour, but some may take longer (up to 7 days).
                   Some may require Proof of Association or be
                   declined. Declined deposits can be recovered
                   publicly via ragequit.
  Custom RPC?       Pass --rpc-url on any command, or save per-chain overrides in
                   ~/.privacy-pools/config.json under "rpcOverrides": { "<chainId>": "https://..." }.

------------------------------------------------------------------------------------------------------------------------
Exit Codes:
  0  Success
  1  Unknown/general error
  2  Input/validation error
  3  RPC/network error
  4  Setup required or incomplete local configuration
  5  Relayer error
  6  Proof generation error
  7  Contract revert
  8  ASP error

------------------------------------------------------------------------------------------------------------------------
Using CLI with an Existing Website Account:
  If you already use privacypools.com, you can access the same account from the CLI:
  1. Export your 12/24-word recovery phrase from the website.
  2. Run: privacy-pools init --recovery-phrase-file ./recovery.txt (or: cat recovery.txt | privacy-pools init --recovery-phrase-stdin)
  3. Set your signer key: export PRIVACY_POOLS_PRIVATE_KEY=0x...
  4. Run: privacy-pools accounts  (syncs onchain state automatically)
  If the website account was created before the 2024 security update, run privacy-pools migrate status to inspect legacy readiness on CLI-supported chains. Review beta or website-only legacy migration surfaces in the website.

------------------------------------------------------------------------------------------------------------------------
Terminology:
  Recovery phrase          24-word mnemonic by default in the CLI; imports may be 12/24 words.
  Signer key               Private key that pays gas and sends transactions.
  Pool Account             Your onchain deposit, tracked for withdrawal or ragequit recovery.
  0xBow ASP status  approved (withdraw ready), pending (waiting),
                                   poa_required (Proof of Association needed),
                                   declined (ragequit available), unknown (unresolved).
  Relayed withdrawal       Privacy-preserving withdrawal via a relayer (recommended).
  Direct withdrawal        Non-private withdrawal; links deposit and withdrawal onchain.
  Ragequit    Self-custody guarantee. Public, irreversible recovery to original deposit address.
